# CSV Import Partner Specification - Active

## Purpose

This document defines how partner ticket CSV files must be imported during the V2 release.

Partner files are generated by several customer systems and may contain blank lines, comment rows, inconsistent whitespace, mixed-case priorities, duplicate external IDs, and pipe-delimited tags.

## Reviewed Function

Public function:

`supportdesk.importers.import_tickets_csv(csv_text)`

## Required Columns

The CSV input uses these columns:

- `external_id`
- `subject`
- `priority`
- `tags`
- `customer_id`

Additional columns may be ignored.

## Row Handling

The importer must:

- Ignore blank rows.
- Ignore rows where the first non-space character of the `external_id` field is `#`.
- Trim field values.
- Skip rows without a usable external ID.
- Normalize ticket data using the active ticket normalization rules.
- Return a dictionary keyed by external ID.

## Duplicate Handling

If the same external ID appears multiple times, the later row replaces the earlier row.

This is intentional. Partner systems sometimes send corrected rows later in the file.

Example:

```csv
external_id,subject,priority,tags,customer_id
A-1,First subject,HIGH,vip|network,C1
A-1,Replacement,low,billing|vip,C3
```

Expected result:

- key A-1 exists once
- subject is Replacement
- priority is low
- customer_id is C3

## Tag Handling

The tags field uses a pipe delimiter:

```csv
vip | network | VIP
```

Tags must be passed through the same normalization rules used by ticket normalization:

- trim
- lowercase
- remove empty tags
- deduplicate
- sort

## Priority Handling

Priority must also follow active ticket normalization rules.

Unknown priority becomes `normal`.

## Reason for Replacement Semantics

Partner feeds are append-corrected. The final occurrence of an external ID represents the most recent state.
